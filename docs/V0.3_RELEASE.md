# WSVPN V0.3 Release Notes

## Overview
WSVPN V0.3 adds multi-client support with UUID-based authentication, health monitoring endpoint, and configuration hot reload capabilities.

## New Features

### Phase 1: Multi-Client Support ✅
- **UUID-based Authentication**: Clients authenticate via UUID in WebSocket path (`/ws/{uuid}`)
- **Static IP Allocation**: Each UUID is assigned a fixed IP from `clients.json`
- **Client Manager**: Thread-safe client management with RWMutex protection
- **Unauthorized Rejection**: Unknown UUIDs are rejected at connection time

**Files Added:**
- `src/server/client_manager.go` - ClientManager struct with UUID→IP mapping

**Files Modified:**
- `src/server/main.go` - Extract UUID from path, use ClientManager for IP assignment
- `src/client/main.go` - Add UUID field to config, use UUID in WebSocket URL

### Phase 2: Health Endpoint ✅
- **GET /ws/health?token=<admin_token>** - Returns comprehensive server status
- **Metrics Tracking**: Bytes in/out, packets in/out, client count, uptime
- **System Metrics**: Goroutines, memory usage, CPU count, Go version
- **Connected Client Details**: List of active clients with connection times

**Files Added:**
- `src/server/handlers.go` - HandleHealth() and HandleConfigReload() endpoints
- `src/server/metrics.go` - Metrics struct with atomic counters

**Files Modified:**
- `src/server/main.go` - Integrated metrics tracking, register health endpoints

### Phase 3: Config Hot Reload ✅
- **SIGHUP Signal Handler**: Reload configuration without restart
- **Atomic Config Swap**: Uses `atomic.Pointer[Config]` for thread-safe updates
- **HTTP Reload Endpoint**: POST /ws/reload?token=<admin_token> for remote reload
- **Obfuscation Toggle**: Enable/disable obfuscation at runtime

**Files Modified:**
- `src/server/main.go` - setupSignalHandlers() for SIGHUP, atomic config storage

### Phase 4: Build & Deploy ✅
- Updated build script supports new file structure
- Both server and client build successfully
- Ready for deployment to ts.vps (10.1.0.252)

## Configuration Files

### server.json
```json
{
  "name": "wsvpn0",
  "network": "10.9.1.0/24",
  "server_ip": "10.9.1.1",
  "listen_addr": ":8080",
  "websocket_path": "/ws/",
  "clients_file": "config/clients.json",
  "log_level": "info",
  "obfuscation": true,
  "admin_token": "<32-char-random-token>"
}
```

### clients.json
```json
{
  "clients": [
    {"uuid": "device-phone-001", "ip": "10.9.1.2", "name": "Joel-iPhone", "enabled": true},
    {"uuid": "device-laptop-002", "ip": "10.9.1.3", "name": "Joel-MacBook", "enabled": true}
  ],
  "network": "10.9.1.0/24",
  "next_dynamic_ip": 50
}
```

### client.json
```json
{
  "name": "wsvpn-client",
  "client_ip": "10.9.1.2",
  "server_url": "ws://10.1.0.252:8080",
  "uuid": "device-phone-001",
  "reconnect": true,
  "log_level": "info",
  "obfuscation": true
}
```

## Usage

### Health Check
```bash
curl "http://10.1.0.252:8080/ws/health?token=<admin_token>"
```

### Config Reload (HTTP)
```bash
curl -X POST "http://10.1.0.252:8080/ws/reload?token=<admin_token>"
```

### Config Reload (Signal)
```bash
kill -SIGHUP $(pgrep wsvpn-server)
```

### Build
```bash
./scripts/build.sh all
```

### Deploy
```bash
./scripts/build.sh deploy-all
```

## API Changes

### WebSocket Path
- **V0.2**: `/ws/` (no authentication)
- **V0.3**: `/ws/{uuid}` (UUID-based authentication)

### Health Endpoint
New endpoint: `GET /ws/health?token=<admin_token>`

## Security Notes
- Admin token must be kept secret
- UUIDs should be unique per device
- Consider using HTTPS/WSS in production
- Admin token validation required for health/reload endpoints
