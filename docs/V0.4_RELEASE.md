# WSVPN V0.4 Release Notes

**Version:** V0.4  
**Status:** Ready for Deployment  
**Release Date:** 2026-03-03

---

## Overview

WSVPN V0.4 adds HTTP/3 (QUIC) support through Nginx configuration updates, providing improved connection performance while maintaining full backward compatibility with V0.3 clients.

---

## What's New

### HTTP/3 (QUIC) Support

**Architecture:**
```
Client → Nginx (QUIC/UDP :443) → WSVPN Server (WebSocket/TCP :8180)
```

**Benefits:**
- **Faster Handshake:** 1-RTT vs 3-RTT (60% reduction)
- **Better Loss Recovery:** QUIC forward error correction
- **Connection Migration:** Maintain connection across IP changes
- **No Head-of-Line Blocking:** Multiplexed streams

**Implementation:**
- Nginx configuration update only
- Zero code changes to WSVPN server/client
- Full backward compatibility with V0.3

---

## Configuration Changes

### Nginx Configuration

**File:** `/etc/nginx/sites-available/app.glidesky.org`

```nginx
server {
    # TCP :443 (HTTP/2, backward compatible)
    listen 443 ssl;
    
    # UDP :443 (QUIC/HTTP/3) ← NEW
    listen 443 quic;
    
    server_name app.glidesky.org;
    
    ssl_certificate /home/sq/.ssl/glidesky/cert.pem;
    ssl_certificate_key /home/sq/.ssl/glidesky/key.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    
    # HTTP/3 support announcement ← NEW
    add_header Alt-Svc 'h3=":443"; ma=86400';
    
    # WebSocket proxy (unchanged)
    location ~ "^/ws/([a-zA-Z0-9_-]{8,64})$" {
        proxy_pass http://127.0.0.1:8180;
        proxy_read_timeout 3600s;
    }
}
```

### WSVPN Configuration

**No changes required!** V0.3 configuration works unchanged.

## Deployment Steps

### Step 1: Verify Nginx Version

```bash
ssh sq@10.1.0.252 "nginx -v"
```

**Requirement:** Nginx 1.25+ or OpenResty 1.21+ with QUIC patch

If version is too old:
```bash
sudo apt-get update
sudo apt-get install nginx
```

### Step 2: Update Nginx Configuration

```bash
# Backup current config
ssh sq@10.1.0.252 "sudo cp /etc/nginx/sites-available/app.glidesky.org /etc/nginx/sites-available/app.glidesky.org.bak"

# Deploy new config (copy from workspace)
scp config/nginx-app.glidesky.org.conf sq@10.1.0.252:/tmp/nginx-new.conf
ssh sq@10.1.0.252 "sudo cp /tmp/nginx-new.conf /etc/nginx/sites-available/app.glidesky.org"

# Test configuration
ssh sq@10.1.0.252 "sudo nginx -t"

# Reload Nginx
ssh sq@10.1.0.252 "sudo nginx -s reload"
```

### Step 3: Verify HTTP/3 Support

```bash
# Check listening ports
ssh sq@10.1.0.252 "ss -tlnp | grep 443 && ss -ulnp | grep 443"

# Expected output:
# LISTEN 0  511  *:443  *:*  users:(("nginx",...))  # TCP
# LISTEN 0  511  *:443  *:*  users:(("nginx",...))  # UDP

# Check Alt-Svc header
curl -skI https://app.glidesky.org | grep Alt-Svc

# Expected output:
# Alt-Svc: h3=":443"; ma=86400
```

### Step 4: Test Client Connections

```bash
# V0.3 client (WebSocket/TCP) - should still work
ssh -p2200 sq@10.1.0.252 "cd ~/wsvpn-test && ./wsvpn-client &"
ssh -p2200 sq@10.1.0.252 "ping -4 -c 5 10.9.1.1"

# HTTP/3 capable client - will automatically use QUIC
# (Browser or curl with --http3 flag)
curl --http3 -skI https://app.glidesky.org
```

## Troubleshooting

### Issue: Nginx fails to start

```bash
# Check error log
ssh sq@10.1.0.252 "sudo tail -20 /var/log/nginx/error.log"

# Verify QUIC support
ssh sq@10.1.0.252 "nginx -V 2>&1 | grep -i quic"
```

If Nginx doesn't support QUIC, upgrade to 1.25+ or use OpenResty.

### Issue: UDP :443 not listening

```bash
# Check firewall
ssh sq@10.1.0.252 "sudo ufw status | grep 443"

# Allow UDP if needed
ssh sq@10.1.0.252 "sudo ufw allow 443/udp"
```

### Issue: Alt-Svc header missing

```bash
# Verify Nginx config
ssh sq@10.1.0.252 "sudo grep -A5 'listen 443 quic' /etc/nginx/sites-available/app.glidesky.org"

# Check if add_header is present
ssh sq@10.1.0.252 "sudo grep 'Alt-Svc' /etc/nginx/sites-available/app.glidesky.org"
```
