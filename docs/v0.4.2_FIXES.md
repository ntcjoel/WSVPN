## Critical Fixes

### 1. WebSocket Concurrent Write Crash 🔴

**Symptom:** Client panics during upload + heartbeat overlap.

**Root Cause:**
- `forwardToServer()` calls `c.conn.WriteMessage()` in a loop
- `irregularHeartbeat()` calls `c.conn.WriteMessage()` every 30-90s
- gorilla/websocket explicitly states: "Connections support one concurrent reader and one concurrent writer"
- Concurrent writes → panic

**Fix:**
```go
type Client struct {
    // ...
    wsWriteMu sync.Mutex // Protect WebSocket writes
}

// In forwardToServer():
c.wsWriteMu.Lock()
err := c.conn.WriteMessage(websocket.BinaryMessage, send)
c.wsWriteMu.Unlock()

// In irregularHeartbeat():
c.wsWriteMu.Lock()
err := c.conn.WriteMessage(websocket.PingMessage, nil)
c.wsWriteMu.Unlock()
```

**Impact:** Eliminates panic during concurrent write scenarios.

### 2. Routing Loop - Incorrect netsh Parameters 🔴

**Symptom:** Network瘫痪 immediately after VPN connection, CPU spikes.

**Root Cause:**
```go
// WRONG: "0.0.0.0" is not a valid interface name
runNetsh("interface", "ipv4", "add", "route", c.serverIP+"/32", "0.0.0.0", "metric=1")
```

- `netsh interface ipv4 add route` syntax: `<prefix> <interface> <nexthop>`
- "0.0.0.0" used as interface name → command fails silently
- Only Warning log, no error returned
- Exclusion route NOT added
- Traffic to VPN server routed into TUN → infinite loop

**Fix:**
```go
// Step 1: Get physical interface and gateway
func getPhysicalRouteInfo(serverIP string) (string, string, error) {
    cmd := exec.Command("route", "print", serverIP)
    out, _ := cmd.CombinedOutput()
    // Parse output to find gateway and interface
}

// Step 2: Use route command (more reliable)
func runRouteAdd(destination, mask, gateway, interface_ string) error {
    cmd := exec.Command("route", "-p", "add", destination, "mask", mask, gateway, "metric", "1", "if", interface_)
}

// Step 3: Fallback to netsh if route fails
if err := runRouteAdd(...); err != nil {
    log.Printf("Warning: route add failed, trying netsh...")
    runNetsh(...) // Fallback
}
```

**Impact:** Prevents routing loop, ensures stable connectivity.

### 3. Goroutine Leak from errCh Buffer Exhaustion 🟠

**Symptom:** After network disconnect, some goroutines never exit.

**Root Cause:**
```go
c.errCh = make(chan error, 2) // Buffer = 2

// 3 goroutines can send errors:
go c.forwardToServer()   // May send error
go c.forwardFromServer() // May send error
go c.irregularHeartbeat() // May send error

// In waitForStop():
err := <-c.errCh // Returns on first error
return err // But 3rd goroutine blocks on c.errCh <- err
```

- 3 goroutines may send errors simultaneously
- Buffer only holds 2
- 3rd goroutine blocks forever on `c.errCh <- err`
- Goroutine leak

**Fix:**
```go
// Increase buffer to match number of error-sending goroutines
c.errCh = make(chan error, 3)

// Non-blocking write
func (c *Client) sendError(err error) {
    select {
    case c.errCh <- err:
    default:
        // Channel full, error already sent
    }
}
```

**Impact:** All goroutines exit cleanly on connection loss.

### 4. log.Fatalf Prevents Reconnect 🟠

**Symptom:** Client exits immediately on first connection failure, ignoring `reconnect: true`.

**Root Cause:**
```go
func connectCmd() {
    // ...
    assignedIP, err := client.connect()
    if err != nil {
        log.Fatalf("Connection failed: %v", err) // Calls os.Exit(1)
    }
    // Never reaches waitForStop() where reconnect logic lives
}
```

- `log.Fatalf()` → `os.Exit(1)` → process terminates
- Reconnect logic in `waitForStop()` never executed

**Fix:**
```go
func connectCmd() {
    for {
        if err := client.runConnection(); err != nil {
            if !cfg.Reconnect {
                log.Printf("Reconnect disabled, exiting: %v", err)
                break
            }
            log.Printf("Connection lost: %v", err)
            log.Println("Reconnecting in 5 seconds...")
            time.Sleep(5 * time.Second)
            client.stop()
            client = &Client{cfg: cfg} // Reset and retry
        } else {
            break // Clean exit
        }
    }
}

func (c *Client) runConnection() error {
    // All connection logic here
    // Returns error on failure (doesn't exit)
}
```

**Impact:** Reconnect now works as expected.

## Improvements

### 5. Configurable QUIC SNI

**Before:**
```go
tlsCfg := &tls.Config{
    ServerName: "app.glidesky.org", // Hardcoded
}
```

**After:**
```go
// Config struct
type Config struct {
    // ...
    QUICSNI string `json:"quic_sni"`
}

// Usage
sni := c.cfg.QUICSNI
if sni == "" {
    sni = "app.glidesky.org" // Default
}
```

**Benefit:** Supports domain fronting and custom certificates.

---

### 6. Removed Unnecessary sync.Pool

**Before:**
```go
var packetPool = sync.Pool{
    New: func() interface{} { return make([]byte, 2048) },
}

func forwardToServer() {
    buf := packetPool.Get().([]byte)
    defer packetPool.Put(buf)
    // ...
}
```

**After:**
```go
func forwardToServer() {
    buf := make([]byte, 2048)
    // ...
}
```

**Rationale:** `sync.Pool` is for high-frequency short-lived objects. Here, buffer lifetime equals connection lifetime, so pool provides no benefit.

**Benefit:** Simpler code, same performance.
